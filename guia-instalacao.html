1

🟨server.jsservidor / proxy

2
Criar conta no GitHub e subir os arquivos
10 min
▼
O que é o GitHub? É um site gratuito que guarda os arquivos do seu sistema, igual ao Google Drive mas para código. O Render vai buscar os arquivos direto de lá.
1
Crie uma conta gratuita no GitHub
Acesse github.com → clique em Sign up → preencha e-mail, senha e nome de usuário → confirme o e-mail que chegar na sua caixa.
2
Crie um repositório (pasta) para o sistema
Depois de entrar no GitHub:

→ Clique no + no canto superior direito → New repository
→ Em Repository name escreva: expedicao-bling
→ Deixe marcado Public
→ Marque a caixinha "Add a README file"
→ Clique no botão verde Create repository
3
Faça upload do server.js e package.json
Dentro do repositório que você criou:

→ Clique em Add file → Upload files
→ Arraste ou selecione o server.js e o package.json
→ Lá embaixo, clique em Commit changes
⚠ Atenção: o index.html ainda NÃO vai aqui. Ele vai dentro de uma pasta chamada public. Siga o passo 4.
4
Crie a pasta "public" e faça upload do index.html
O GitHub não deixa criar uma pasta vazia, então o truque é criar um arquivo dentro dela:

→ Clique em Add file → Create new file
→ No campo do nome do arquivo, escreva: public/index.html
   (quando você digitar a barra /, o GitHub vai criar a pasta automaticamente)
→ No campo de conteúdo abaixo, cole todo o conteúdo do index.html que você baixou
→ Clique em Commit changes
🔎 Como abrir o index.html para copiar: clique com botão direito no arquivo → Abrir com Bloco de Notas → Ctrl+A (selecionar tudo) → Ctrl+C (copiar) → cole no GitHub.
5
Confira se ficou assim
Seu repositório deve ter essa estrutura:
📁public/pasta
🌐public/index.htmlsistema visual
🟨server.jsservidor
📋package.jsonconfigurações
📄README.mdcriado automaticamente
✅ Se ficou assim, o GitHub está pronto! Próxima etapa: colocar no ar.
3
Criar conta no Render e colocar no ar
10 min
▼
O que é o Render? É um serviço gratuito que pega seus arquivos do GitHub e coloca no ar na internet, gerando um link que funciona no celular de qualquer lugar.
1
Crie uma conta gratuita no Render
Acesse render.com → clique em Get Started for Free → escolha Sign up with GitHub (mais fácil, já conecta direto).
2
Crie um novo "Web Service"
Dentro do Render, no painel principal:

→ Clique em New + (botão roxo no canto superior direito)
→ Clique em Web Service
3
Conecte ao seu repositório do GitHub
Na tela seguinte:

→ Clique em Connect account (se pedido) para autorizar o GitHub
→ Vai aparecer seu repositório expedicao-bling na lista
→ Clique em Connect ao lado dele
4
Configure o serviço
Preencha os campos assim:
CampoO que colocar
Nameexpedicao-bling (ou qualquer nome)
RegionSão Paulo (South America) se disponível, senão deixe padrão
Branchmain (já deve estar preenchido)
Build Commandnpm install
Start Commandnode server.js
Instance TypeFree (plano gratuito)
💡 Se o campo Build Command já tiver preenchido com outro valor, apague e coloque npm install.
5
Clique em "Create Web Service"
Role a página até o fim → clique no botão Create Web Service.

O Render vai começar a instalar e rodar o sistema. Isso leva de 2 a 5 minutos. Você vai ver logs na tela — quando aparecer "Live" em verde ou a mensagem "Servidor rodando na porta 3000", está pronto.
6
Copie o link do seu sistema
No topo da página do Render aparece o link do seu sistema, algo como:

https://expedicao-bling.onrender.com

Clique nele — vai abrir o sistema de expedição! 🎉
✅ Pronto! O sistema está no ar. Agora só falta conectar ao Bling.
4
Pegar o token do Bling e configurar
10 min
▼
O que é o token? É uma senha especial que o Bling gera para deixar outros sistemas acessarem seus dados. Sem ele, o sistema não consegue puxar os pedidos.
1
Crie um aplicativo no Bling
Dentro do Bling, no menu superior:

→ Clique no ícone de grade/apps (canto superior direito) → Área do Integrador
→ Ou acesse direto: bling.com.br/cadastro.aplicativos.php

→ Clique em + Novo aplicativo
→ Nome: Expedição (qualquer nome)
→ Em Redirect URI coloque: https://expedicao-bling.onrender.com/callback (substituindo pelo seu link real)
→ Salve — o Bling vai gerar um Client ID e um Client Secret. Guarde os dois.
💡 O nome e URL exatos variam um pouco dependendo da versão do seu Bling. Se travar, tire um print e me mande no chat que eu te ajudo.
2
Gere o Access Token (senha de acesso)
O Bling usa OAuth 2.0. O fluxo mais rápido para uso interno é:

→ Dentro do aplicativo criado, clique em Autorizar ou Gerar Token
→ Faça login com seu usuário Bling quando pedido
→ O Bling vai retornar um access_token — copie esse valor.

Importante: O token expira em 6 horas. Para uso diário, você vai precisar renovar ou configurar o refresh_token (posso ajudar com isso depois).
⚠ O token é como uma senha — não compartilhe com ninguém além dos funcionários que vão usar o sistema.
3
Configure o token no sistema
No sistema de expedição (o link do Render):

→ Clique na aba ⚙ Config
→ Cole o token no campo Access Token (Bearer)
→ O campo URL do Proxy já vai estar preenchido automaticamente
→ Clique em Salvar

O botão ● Bling ON deve ficar verde. Agora clique em Buscar no Bling — os pedidos de hoje devem aparecer!
5
Como usar no dia a dia
referência
▼
📱
Abrir no celular
O link do Render abre em qualquer celular com Chrome. Para deixar como atalho na tela inicial:

→ Abra o link no Chrome → toque nos 3 pontinhos (⋮) → Adicionar à tela inicial
☀
Todo dia de manhã — Romaneio
→ Abra o sistema → aba Romaneio
→ Clique em Buscar no Bling
→ Os pedidos do dia aparecem divididos por marketplace (Shopee, ML, Amazon...)
→ Isso vem direto do campo Serviço / Logística do Bling
📦
Na hora de embalar — aba Coleta
→ Aba 📷 Coleta
→ Selecione a transportadora que vai chegar
→ Clique em Iniciar Câmera (ou use leitor USB na caixinha de entrada)
→ Aponte para o código de barras da etiqueta — o sistema lê, tira foto automática e registra
→ A barra de progresso mostra quantos faltam
→ Ao terminar tudo, clique em Confirmar entrega ao motorista
→ O status atualiza no Bling automaticamente
🚫
Se biper o pacote errado
→ Aparece um aviso vermelho grande: "MARKETPLACE ERRADO!"
→ O sistema mostra de qual transportadora é o pacote
→ Não registra — o estoquista coloca de volta no carrinho certo
📸
Evidências — aba Dia
→ Aba Dia tem todas as fotos das etiquetas bipadas com data e hora
→ Se tiver dúvida depois ("foi entregue mesmo?"), é só olhar aqui
→ Dá para exportar CSV com todo o histórico do dia
⚠ Sobre o plano gratuito do Render: No plano grátis, o servidor "dorme" depois de 15 minutos sem uso e demora ~30 segundos para acordar na primeira visita do dia. Para uso na expedição (onde você abre todo dia de manhã), isso não é problema. Se quiser sem esse delay, o plano pago custa ~US$ 7/mês.
💡 Token expirando? O token do Bling dura 6 horas. Se aparecer erro "401 Unauthorized", é só ir em ⚙ Config e colar um token novo. Me peça no chat que posso criar um sistema de renovação automática.
🆘 Travou em algum passo?
Manda print ou descreve onde travou no chat — vou te ajudar a resolver. Os passos mais comuns onde as pessoas travam são:

• Estrutura de pastas no GitHub — a pasta public/ precisa existir
• Token do Bling — o fluxo OAuth pode confundir, mas resolvemos juntos
• Câmera no celular — precisa ser Chrome e o site precisa ser HTTPS (o Render já faz isso automaticamente)
